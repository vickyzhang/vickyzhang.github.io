workerman 浏览器消息推送

公司直播室网站，在直播室留言区需要使用socket推送功能。 我去公司之前，同事们已经布置好workerman的环境，没得选择我使用了workerman 作为我的socket推送服务器。 直播室聊天室相较于其他聊天室多了一个消息审核功能。也就是我发出的消息只能先推送到管理员处，由管理员审核后才能推送给所有观众。 所以实现的流程 发送-入库-推送给管理员-审核/删除-推送给所有人。 由于workerman提供的手册显示，他的浏览器推送支持推送给一个人，和推送给所有人，但是我们同时有多个直播室管理员在线，消息应该是推送给所有管理员，任一管理员审核后推送给所有人。所有在第一次使用workerman推送给所有管理员的时候，我在PHP代码中用了循环推送。测试使用时一切正常。只是推送速度有点慢，但是功能是没有问题。 但是每到晚上8点左右，迎来了直播室观看高峰期，公屏上互动信息变多，推送消息频率增加。这个时候，会发现推送给管理员的数据会部分丢失。并且打开浏览器调试模式会发现浏览器一直发送socket请求。 在确认代码无误的情况下，我多次找问题，实验都以失败告终。 最后我觉得把循环推送管理员的地方做修改，在workerman的start.php循环，而不是在我的代码里循环请求推送。感天动地，问题终于得到解决。 原来是因为workerman是用的curl请求推送，我在php代码循环请求curl链接，导致服务器堵塞。而我改在workerman服务端循环，相关于我只发起一个请求，他在服务器端多次推送。 ``` //前端js ``` ``` //php推送代码 "publish", "content" => "这个是推送的测试数据", "to" => $to_uid, ); $ch = curl_init (); curl_setopt ( $ch, CURLOPT_URL, $push_api_url ); curl_setopt ( $ch, CURLOPT_POST, 1 ); curl_setopt ( $ch, CURLOPT_HEADER, 0 ); curl_setopt ( $ch, CURLOPT_RETURNTRANSFER, 1 ); curl_setopt ( $ch, CURLOPT_POSTFIELDS, $post_data ); $return = curl_exec ( $ch ); curl_close ( $ch ); var_export($return); ``` ``` //服务端的start.php $sender_io->on('workerStart', function(){ // 监听一个http端口 $inner_http_worker = new Worker('http://0.0.0.0:2121'); // 当http客户端发来数据时触发 $inner_http_worker->onMessage = function($http_connection, $data){ $_POST = $_POST ? $_POST : $_GET; // 推送数据的url格式 type=publish&to=uid&content=xxxx switch(@$_POST['type']){ case 'publish': global $sender_io; $to = @$_POST['to']; $_POST['content'] = htmlspecialchars(@$_POST['content']); // 有指定uid则向uid所在socket组发送数据 if($to){ foreach(explode(',', $to) as $v){ $sender_io->to($v)->emit('new_msg', $_POST['content']); } // 否则向所有uid推送数据 }else{ $sender_io->emit('new_msg', @$_POST['content']); } // http接口返回ok return $http_connection->send('ok'); } return $http_connection->send('fail'); }; // 执行监听 $inner_http_worker->listen(); }); ```
